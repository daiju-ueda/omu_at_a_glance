# 評価軸強化（FWCI合計＋指標別学内順位） 設計書

作成日: 2026-07-02
ステータス: 承認済み（著者数補正版FWCI合計は不採用と決定）
前提: Phase 1＋Tier 1＋KAKEN＋比較ビューが main にマージ済み。

## 目的

1. **FWCI合計（ΣFWCI）** を追加し**デフォルトソートにする**。根拠: FWCI平均は論文数が少ない研究者ほど1本の高FWCI論文（偶然の共著等）に支配される。合計は量×質で継続的な研究活動を反映する。平均・中央値は列として残す（量バイアスの相互チェック）
2. **指標別学内順位**: 研究者詳細ページの各指標に「学内◯位 / 母数」を併記し、多面的な立ち位置を見えるようにする

## 変更内容

### メトリクス

- `researcher_metrics.fwci_total: float (default 0)` — ウィンドウ内worksのFWCI（非NULL）の合計、小数4桁丸め。該当0本なら0
- compute_metrics で集計（fwci_mean/median と同じ `fwcis` リストから `sum`）

### ランキング（`/`）

- `SORT_COLUMNS` に `fwci_total` を追加（8キー）
- **デフォルトソートを `fwci_mean` → `fwci_total` に変更**（queries.ranking のデフォルト引数、app のフォールバック、テンプレートのソート済みマーカーの3箇所）
- 列: 「被引用(補正)」の直後に **FWCI合計**（ソート可）を追加。既存のFWCI平均/中央値列は維持

### 研究者詳細（`/researchers/{id}`）

- メトリクスカードに **FWCI合計** を追加（FWCI平均の前）
- **学内順位表示**: 以下8指標の値の下に「学内◯位」を小さく併記
  - 3年論文数 / 総被引用数 / 被引用(補正) / FWCI合計 / FWCI平均 / top10%論文数 / top1%論文数 / 科研費配分総額
- 順位定義: 母数 = `works_count_3y >= 1` の研究者（現在3,338人）。順位 = `1 + COUNT(該当指標が本人より大きい)`（同値は同順位、上位詰め）。本人の指標がNULL、または本人が母数外（works 0本）の場合は順位非表示
- 実装: `queries.metric_ranks(session, researcher_id) -> dict[str, tuple[int, int]] | None`（指標キー → (順位, 母数)。詳細ページ表示時にオンザフライ計算。3,876行のテーブルに8回のCOUNT——十分軽い）

### 比較ビュー（`/compare`）

- インパクトグループに **FWCI合計** 行を追加（FWCI平均の前、ハイライト対象）

### DB反映

確立済みの運用: `rm db/researchers.db*` → full sync で再構築。

## 表示上の注意

- FWCI合計は多作なほど有利（量バイアス）。平均・中央値・順位を併記することで相互に補正する。フッター注記は変更なし

## テスト

- metrics: fwci_total の合計値（NULL除外）、0本→0
- queries: ranking デフォルトが fwci_total 降順、metric_ranks の順位計算（同値同順位・NULL非表示・母数外None）
- web: デフォルトソートの表示順、FWCI合計列、詳細の順位表示（「学内◯位」文字列）、比較ビューの行追加
- 実DB再構築後のスポットチェック

## スコープ外（YAGNI）

- 著者数補正FWCI合計（ユーザー判断で不採用）
- 順位のパーセンタイル表示、部局内順位（サブプロジェクトB=部局統合後に検討）
