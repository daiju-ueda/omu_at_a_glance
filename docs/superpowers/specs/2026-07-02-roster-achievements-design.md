# 公式総覧の個人ページ実績（受賞・著書・講演・委員歴） 設計書

作成日: 2026-07-02
ステータス: 承認済み
前提: 部局統合＋重複著者統合まで main にマージ済み。researchmapスクレイピングは不採用（robots.txt Disallow）と決定済み——同等データを大学自身の公開DBから取る。

## 実査結果（2026-07-02、磐下徹氏のページで確認）

- 個人ページURL: `/html/{profile_id}_ja.html`（profile_idはroster取得済み、約290KB/ページ）
- 実績はページ内にインラインで存在（JS展開はアコーディオンのみ）
- コンテナ: `div#jusho`（受賞歴）、`div#chosho`（書籍等出版物）、`div#knkyu_prsn`（講演・口頭発表等）、`div#gkkai_iinkai`（委員歴（学外））
- エントリ構造: コンテナ内 `ul > li`、タイトルは `p.title`、年月・機関は後続の `p.contents`（例:「2010 日本歴史学会」「2023年12月 大阪公立大学」「2016年11月」）

## 収集・保存

- クロール: roster同期の後、roster全員の個人ページを取得（既存RosterClient・0.5秒間隔、1,382ページ≒12分。週次syncに追加）
- **roster_achievements**: `id`(PK autoincrement), `profile_id`(index), `category`("award"|"book"|"presentation"|"committee"), `title`, `year: int|None`（p.contentsから最初の `(19|20)\d{2}` を抽出）, `detail`(年月機関行の原文), `updated_at`
- 全洗い替え。ただし**取得成功ページがroster人数の50%未満なら洗い替えせず既存保持＋警告**（全消しガード）。ページ単位の失敗・パース不能セクションはスキップ＋警告
- パース失敗時もページ内の他セクションは収集を続行

## メトリクス（researcher_metrics に4列追加）

`awards_count` / `books_count` / `presentations_count` / `committee_count`（int, default 0）
— 名寄せ済み（roster.matched_researcher_id → 正準researcher）の実績エントリ数を compute_metrics で集計。**全期間値**（論文系の3年窓と異なる）。

## 表示

- 研究者詳細: メトリクスカードに4項目（ラベルに「全期間」を含める: 例「受賞（全期間）」）。カードの下に**受賞歴リスト**（賞名・年、年降順、全件——受賞は件数が少ないため上限なし）
- 比較ビュー: 新グループ「実績（全期間・公式総覧）」に4行（ハイライト対象）
- フッター注記に追記: 「受賞・著書・講演・委員歴は公式総覧収録分（全期間・名寄せ済み研究者のみ）」
- ランキングには列を追加しない（YAGNI）

## テスト

- パーサ: 実構造fixture（4セクション、年抽出「2010」「2023年12月」、詳細アコーディオン無視、空セクション、タイトル欠落liスキップ）
- 同期: 50%ガード、ページ失敗スキップ、洗い替え冪等
- メトリクス: 名寄せ経由の件数集計（未マッチrosterは集計外、エイリアス経由は正準へ）
- web: カード4項目・受賞リスト・比較4行・注記
- 実クロール後: 件数分布・著名研究者のスポットチェック

## スコープ外（YAGNI）

- 取得学位・研究キーワード・担当授業の指標化（データはページにあるが比較軸として弱い）
- 講演の招待/一般の区別（詳細アコーディオン内で欠損が多い）
- 受賞リスト以外の実績リスト表示（著書等は件数のみ）
