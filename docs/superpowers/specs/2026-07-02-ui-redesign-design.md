# UI/UX全面改善（可読性・情報設計） 設計書

作成日: 2026-07-02
ステータス: 承認済み（「ウェブを見やすくしといて」の指示による）
前提: 実績統合まで main にマージ済み。サーバーレンダリング＋CSS1枚＋素JSという構成は**維持**する（DX: ビルドなし・トークンはCSS変数・単一スタイルシート）。

## デザインの立脚点

- 主題: 学術計量の**計器**。読者は研究者本人・研究室・大学関係者。ページの仕事は「公平に・速く・多面的に研究者を読む」
- 美学: 冷たい紙×藍×金（賞状・印章の語彙）。データ密度を敵にせず、階層で制御する
- シグネチャ: **ソート列の分布バー** — ランキング表で現在ソート中の指標セルに、ページ内最大値比の細いバーを敷く。表が同時に分布図になる（サーバー計算・JS不要）
- 動きは最小限（hover遷移のみ、prefers-reduced-motion尊重）。フォントはシステム和文スタック＋数値は `font-variant-numeric: tabular-nums`（Webフォント外部依存なし）

## 共通（base.html / style.css）

- ヘッダ: 藍地（--indigo-deep）。サイト名は「OMU研究者比較」＋小さく「Research Metrics」。ナビ現行4リンク＋現在地の下線
- タイポ: 本文15px/1.65、見出しはウェイトと字間で階層（h2は1.2rem・太字・下罫）。数値セルは tabular-nums・右揃え・桁区切り
- フッター: 注記を「データについて」小見出し付きの箇条書き（現在の1行連結文を分解して可読化。**文言自体は変えない**——センテンス単位で `<li>` に分割）
- フォーカス可視（:focus-visible）、reduced-motion対応、モバイルは表を横スクロールコンテナ化（氏名列sticky）

## ランキング（ranking.html）

- **2段ヘッダ**: 上段にグループ見出し（選択/順位/氏名｜生産性｜インパクト｜資金）を colspan で、下段に現行の列。列順は不変
- **ソート列の強調＋分布バー**: ソート中の列は `th.sorted`（藍下線・太字）、各tdに `.sorted` ＋ 淡藍の地、セル内下端に `--w`%幅のバー（ページ内最大値比。値NULL/0はバーなし）。バー幅はサーバー計算（app側でrowsから算出しテンプレへ）
- 数値列は右揃え・tabular-nums・桁区切り。二次列（FWCI中央値・筆頭・責任）は --muted
- 行hoverで薄藍。順位（#）とチェックボックスは低コントラスト
- コントロール行: 部局セレクト・最低論文数・総件数を1行のツールバーに整理
- ソートマーカー「▼」・全ラベル・「全X人中 Y人を表示」文言は**現行のまま**（テスト互換）

## 研究者詳細（researcher.html）

- **アイデンティティブロック**: 氏名（大・名寄せ済みなら日本語）、下にローマ字/カナ相当（display_name）、部局・職位、外部リンクはチップ型ボタン（OpenAlex / ORCID）
- **メトリクスを4セクションに再編**（dt/ddラベル文字列は不変。並べ替えとセクション見出しの追加のみ）:
  1. インパクト — FWCI合計・FWCI平均・FWCI中央値・総被引用数・被引用(補正)・top10%論文・top1%論文
  2. 生産性 — 3年論文数・論文数(補正)・筆頭著者数・責任著者数・平均著者数・ユニーク共著者
  3. 連携・資金 — 国際共著率・産学連携率・OA率・科研費（代表/分担/配分総額）
  4. 研究者指標・実績（全期間） — h指数・i10指数・2年平均被引用・受賞・著書・講演・委員歴・preprint・データ/SW・主分野
- 主要4指標（FWCI合計・総被引用・3年論文数・科研費配分総額）は大きめの数字。学内順位チップは現行の `.rank` を藍のピル型に
- 受賞歴リストはカード化（年を右肩に）

## 比較（compare.html）

- 1列目（指標名）と研究者ヘッダ行を sticky
- bestセル: 緑地をやめ、**金の左罫＋金地の淡＋太字**（`td.best`）に統一（賞の語彙）
- グループ行は藍の小見出し帯

## 部局間比較（departments.html）

- 数値右揃え・桁区切り、人あたりFWCI合計列を藍で強調（既存strongを置換）、参考表の opacity は維持

## 実装上の制約（テスト互換・DX）

- 既存テストが参照する文字列・クラス（`class="best"`、`class="cmp"`、`id="compare-bar"`、`.rank`、ラベル文言、▼、内訳文言、フッター注記文言）は**変更しない**。フッターは文分割のみ（各センテンスの文字列は保存）
- 追加するサーバーロジックはランキングの「ソート列最大値→バー%」計算のみ（app層・純関数・テスト付き）
- style.css は全面書き換え（トークン→ベース→コンポーネントの順で整理）。テンプレは構造追加（クラス付与・セクション化）のみで文言不変

## テスト

- バー%計算の純関数（最大値・NULL/0・空ページ）
- ランキングに `class="sorted"` と `--w` が現れる／非ソート列に出ない
- 詳細のセクション見出し4つ・アイデンティティブロック
- 既存137テストが**無修正で**全pass（文言・クラス互換の検証を兼ねる）
- 実DBでの目視代替: 主要4ページのcurl＋構造グレップ

## スコープ外（YAGNI）
