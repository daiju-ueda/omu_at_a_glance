# 並列比較ビュー 設計書

作成日: 2026-07-02
ステータス: 承認済み
前提: Phase 1＋Tier 1指標＋KAKEN統合が main にマージ済み。閲覧Webはポート8100で稼働中。

## 目的

研究者2〜4人を横並びで比較する `/compare` ページを追加する（本サービスの中核UX）。選択はランキング・検索結果のチェックボックスから。

## 設計

### 選択UX（ランキング・検索）

- ランキング表・検索結果表の各行の先頭にチェックボックス（`data-id` にopenalex_id、`data-name` に表示名）
- 画面下部に固定バー（初期非表示）: 「比較する（n人）」ボタン＋選択中の氏名表示。チェック数が2〜4のときだけボタン有効
- クリックで `/compare?ids=<id1>,<id2>[,<id3>,<id4>]` へ遷移
- 実装は `web/static/compare.js`（バニラJS 約30行、ビルドなし）。base.htmlで読み込み。JSが無効でも既存ページの閲覧機能は一切損なわれない（プログレッシブエンハンスメント）

### `/compare?ids=` ページ

- `ids` はカンマ区切り。パース規則: 空白除去→重複除去（先勝ち）→存在しないIDは無視→先頭4件に切り詰め
- 有効IDが2未満 → 比較不能メッセージ＋ランキングへの誘導リンク（200で返す。500や404にしない）
- 列=研究者、行=指標の縦長テーブル。行構成:
  - **基本**: 氏名（`name_ja or display_name`、詳細ページへリンク）、主分野、h指数（全期間）、i10指数（全期間）
  - **生産性**: 3年論文数、論文数(補正)、筆頭著者数、責任著者数
  - **インパクト**: 総被引用数、被引用(補正)、FWCI平均、FWCI中央値、top10%論文数、top1%論文数
  - **連携・資金**: 国際共著率、産学連携率、OA率、科研費（代表）、科研費（分担）、科研費配分総額
- **行ごとの最良値ハイライト**: 数値行で最大値のセルに `class="best"`（太字＋薄背景）。全員同値なら付けない。欠損（None）は比較対象外。「最良=最大」で統一（全行とも大きいほど良い指標のみを載せる）
- メトリクス行が無い研究者（metrics未計算）は列ごと「–」表示
- **分野注意**: 選ばれた研究者の `top_subfield`（非NULL）が2種類以上なら、表の上に「主分野が異なります。生の被引用数ではなく FWCI 等の分野正規化済み指標での比較を推奨します」を表示
- 表示フォーマットは既存Jinjaフィルタ（fmt / pct / man）を再利用。フッター注記は共通（base.html）

### クエリ層

- `web/queries.py` に `compare(session, ids: list[str]) -> list[Row]` を追加（Row = Researcher × ResearcherMetrics の outerjoin、**引数の順序を保持**して返す）

## エラー処理

- `ids` 欠如・空・全て無効 → 案内メッセージ（200）
- 5件以上は先頭4件（エラーにしない）

## テスト

- queries.compare: 順序保持・重複除去・不明ID無視・metrics無し研究者のouterjoin
- /compare: 正常表示（2人・4人）、best ハイライトの正当性（最大値セルにclass）、分野相違の注意表示/非表示、有効1人以下のメッセージ、不正ids文字列で200
- チェックボックスとバーの存在（HTMLレンダリング確認。JSの動作自体はサーバーテスト対象外）

## スコープ外（YAGNI）

- レーダーチャート等の可視化（データが揃った後に検討）
- 比較結果の保存・共有短縮URL
- 5人以上の比較
