# 指標説明ページ（/about） 設計書

作成日: 2026-07-02
ステータス: 承認済み（「メトリクスの説明ページも作成しておいて」の指示による）

## 目的

全指標の定義・データソース・計算方法・限界を1ページで説明し、比較の解釈を誤らせない。サイトの「誠実な比較」方針の明文化。

## 設計

- ルート: `GET /about`（DBアクセスは footer 用の `last_synced` のみ）。テンプレート `web/templates/about.html`
- ナビに「指標の説明」を追加（比較と部局の間）。フッターの「データについて」見出しを `/about` へのリンク化
- 構成（6セクション、既存の `.metric-section` / `.notice` スタイルを再利用）:
  1. **このサイトについて** — 目的・対象（OMU所属研究者）・「収録分に基づく」原則
  2. **データソースと更新** — OpenAlex（論文・被引用、CC0）/ KAKEN（科研費）/ 大学公式研究者総覧（部局・職位・氏名・実績）。週次全量更新・直近3年ローリング窓（実績と科研費の扱いの違いも明記）
  3. **研究者の統合と名寄せ** — OpenAlex重複レコードの証拠ベース統合、KAKEN/公式総覧との名寄せ（学内一意のみ）、名寄せ済み人数の現況は書かない（変動するため「一部の研究者は名寄せできていない」と定性で）
  4. **指標の定義** — 詳細ページと同じ4グループ構成の定義テーブル（指標名・定義・出所・窓）。FWCIの説明（分野・出版年正規化、1.0=世界平均）と「なぜFWCI合計が既定ソートか」（平均は論文数が少ないと1本に支配される）を重点的に
  5. **学内順位の読み方** — 母数（3年1本以上）・同値同順位・値0は順位非表示
  6. **限界** — OpenAlex未収録（和文誌・人文系に弱い）/ preprintと出版版の重複計上 / 大規模共著の著者数打ち切り / 実績は本人登録 / 名寄せ未マッチの存在
- 文体: です・ます調、指標名はUIの表記と完全一致（例:「被引用(補正)」）
